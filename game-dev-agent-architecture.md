# 独立游戏开发 Agent 架构设计

> 创建日期：2026-03-27
> 用途：Godot + OpenClaw 独立游戏开发

## 整体架构

```
                        ┌─────────────────────────────────┐
                        │           你 (Boss)              │
                        │   提出想法、做决策、画素材        │
                        └───────────────┬─────────────────┘
                                        │
                        ┌───────────────▼─────────────────┐
                        │      Orchestrator (主控)         │
                        │  - 理解你的意图                  │
                        │  - 拆解任务                      │
                        │  - 分发给专业 Agent              │
                        │  - 整合结果汇报                  │
                        └───────────────┬─────────────────┘
                                        │
            ┌───────────┬───────────────┼───────────────┬───────────┐
            │           │               │               │           │
            ▼           ▼               ▼               ▼           ▼
      ┌──────────┐ ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐
      │ GDScript │ │  Game    │  │ Project  │  │  Asset   │  │  Writer  │
      │  Coder   │ │ Designer │  │ Manager  │  │ Manager  │  │ (可选)   │
      └──────────┘ └──────────┘  └──────────┘  └──────────┘  └──────────┘
```

---

## 各 Agent 职责

### Orchestrator（主控）

**角色**：项目经理 + 翻译

**职责**：
- 接收你的想法
- 翻译成具体任务
- 决定发给哪个 Agent
- 整合多个 Agent 的输出
- 向你汇报进度和结果

**输入**：「我想加个背包系统」
**输出**：拆解 → Game Designer 设计 → GDScript Coder 实现 → Asset Manager 记录素材需求 → 整合汇报

---

### GDScript Coder

**角色**：程序员

**职责**：
| 事 | 不做 |
|----|------|
| 写 GDScript 代码 | 设计玩法 |
| 搭建节点结构 | 决定数值 |
| 实现游戏逻辑 | 画素材 |
| 修复 bug（指定位置） | 项目管理 |
| 代码重构优化 | 写文档 |

**输入**：「实现背包系统，10 格，支持拖拽」
**输出**：GDScript 代码 + 节点结构说明

---

### Game Designer

**角色**：策划

**职责**：
| 事 | 不做 |
|----|------|
| 设计玩法机制 | 写代码 |
| 设计数值（伤害、冷却、掉率...） | 画素材 |
| 设计系统（战斗、背包、任务...） | 管理进度 |
| 竞品调研 | 写剧本 |
| 玩法文档 | |

**输入**：「设计一个背包系统」
**输出**：规格文档（格子数、堆叠规则、拖拽逻辑、UI 布局）

---

### Project Manager

**角色**：进度管理

**职责**：
| 事 | 不做 |
|----|------|
| 记录任务清单 | 写代码 |
| 跟踪完成状态 | 设计玩法 |
| 提醒里程碑 | 画素材 |
| 整理决策记录 | |
| 生成周报 | |

**维护的文件**：
- `project/tasks.md` — 任务清单
- `project/decisions.md` — 决策记录
- `project/roadmap.md` — 里程碑

**输入**：「记录：背包系统开发中」
**输出**：更新任务状态，提醒下一项待办

---

### Asset Manager

**角色**：素材管家

**职责**：
| 事 | 不做 |
|----|------|
| 记录需要什么素材 | 画素材 |
| 记录素材规格（尺寸、帧数） | 导入素材 |
| 检查素材是否缺失 | 设计素材 |
| 生成素材需求文档 | |
| 追踪素材引用关系 | |

**维护的文件**：
- `assets/manifest.md` — 素材清单

**示例记录**：
```markdown
## 角色
- [x] player_idle.png (32x32, 4帧)
- [ ] player_walk.png (32x32, 6帧) ← 缺失
- [ ] player_jump.png (32x32, 3帧) ← 缺失

## 敌人
- [ ] slime_idle.png (16x16, 4帧) ← 缺失
```

**输入**：「代码需要 player_walk 动画」
**输出**：检查清单 → 缺失 → 提醒你画

---

### Writer（可选）

**角色**：文案

**职责**：
| 事 | 不做 |
|----|------|
| NPC 对话 | 设计玩法 |
| 物品描述 | 写代码 |
| 世界观设定 | 画素材 |
| 剧情/任务文案 | |
| UI 文案 | |

**输入**：「给铁匠 NPC 写对话」
**输出**：对话脚本

## 合作流程示例

**你说**：「我想加个史莱姆敌人」

### 第 1 步：Orchestrator 拆解

```
任务：添加史莱姆敌人
├── Game Designer：设计史莱姆属性和行为
├── GDScript Coder：实现史莱姆逻辑
├── Asset Manager：记录所需素材
└── Writer（可选）：如果有对话/描述
```

### 第 2 步：并行执行

**Game Designer 输出**：
```markdown
## 史莱姆规格
- HP: 20
- 移动速度: 50
- 行为: 巡逻 → 发现玩家 → 追击 → 攻击
- 掉落: 史莱姆凝胶 x1-3
```

**Asset Manager 记录**：
```markdown
- [ ] slime_idle.png (16x16, 4帧)
- [ ] slime_move.png (16x16, 4帧)
- [ ] slime_hurt.png (16x16, 2帧)
- [ ] slime_death.png (16x16, 4帧)
```

### 第 3 步：串行执行

**GDScript Coder** 拿到设计文档 → 写代码

### 第 4 步：汇报

**Orchestrator** 整合：
```
史莱姆敌人已添加：
✓ 设计完成：HP 20，巡逻+追击行为
✓ 代码完成：Slime.gd 已创建
⚠ 缺少素材：需要你画 4 组精灵图
```

## 文件结构

```
game-project/
├── project/
│   ├── tasks.md          # 任务清单（Project Manager 维护）
│   ├── decisions.md      # 决策记录
│   └── roadmap.md        # 里程碑
├── design/
│   ├── combat.md         # 战斗系统（Game Designer 维护）
│   ├── inventory.md      # 背包系统
│   └── enemies.md        # 敌人设计
├── assets/
│   └── manifest.md       # 素材清单（Asset Manager 维护）
├── writing/
│   ├── dialogues.md      # 对话（Writer 维护）
│   └── lore.md           # 世界观
└── (Godot 项目文件)
    ├── scenes/
    ├── scripts/
    └── assets/
```

## 总结

| Agent | 角色 | 输入 | 输出 |
|-------|------|------|------|
| Orchestrator | 主控 | 你的想法 | 任务分配 + 结果整合 |
| GDScript Coder | 程序 | 具体功能需求 | 代码 |
| Game Designer | 策划 | 玩法想法 | 设计文档 + 数值 |
| Project Manager | 进度 | 任务状态 | 清单 + 提醒 |
| Asset Manager | 素材 | 代码需求 | 素材清单 + 缺失提醒 |
| Writer | 文案 | 文案需求 | 对话/描述文本 |

**核心原则**：
- 每个 Agent 只做自己擅长的事
- 不越界，不抢活
- 通过 Orchestrator 协调
- 文件是通信媒介

## 美术素材备注

**像素风优势**：
- 门槛相对低
- 资源小，性能好
- 风格统一
- 修改方便

**推荐工具**：
- Aseprite（$20，业界标准）
- Pixelorama（免费，有 Godot 插件版）
- LibreSprite（免费）

**素材来源**：
- Kenney.nl（免费）
- Itch.io（独立画师出售）
- Godot Asset Library（免费）
- OpenGameArt（免费，质量参差）

**精灵图规格建议**：
- 像素密度：16x16 / 32x32 / 48x48
- 限制调色板颜色数量
- 主角约 50-100+ 帧
- 敌人每种 20-50 帧
