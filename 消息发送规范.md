# 消息发送规范

> **适用范围**：所有定时任务信息推送（游戏日报、科技新闻、行业资讯等）
> **创建时间**：2026-03-27
> **维护者**：hasaki

## 📋 基本规则

### 1. 分段规则

**❌ 不要频繁分段**
- 只有当单条消息**超过 4000 字符**时，才使用 `---` 分割
- 不要每条新闻都单独发送
- 不要过度使用 `---`

**✅ 正确做法**
```
超过4000字符的整段内容...

---（4000字符后分割）

后续内容...
```

### 2. 新闻格式规则

**❌ 错误做法**
```
**【重大新闻】**

**1. 标题一**
内容...
📌 来源：GamesIndustry.biz

**2. 标题二**
内容...
📌 来源：GamesIndustry.biz
---
```

**✅ 正确做法**
```
🎮 **游戏行业日报 | 2026年3月27日**

## 【重大新闻】

**1. 微软Project Helix**
GamesIndustry.biz 报道...
来源：GamesIndustry.biz

**2. Epic Games裁员**
Epic Games 因"...
来源：GamesIndustry.biz
```

### 3. 来源信息规则

**❌ 不要这样做**
- 不要在每条新闻上附带完整的来源链接（如 `📌 来源：https://xxx.com`）
- 不要让来源信息占用太多空间

**✅ 要这样做**
- 只要一个来源名称字符串即可（如 `来源：GamesIndustry.biz`）
- 简洁明了

### 4. 标题和分组规则

**消息开头**：使用一个总标题
```
🎮 **游戏行业日报 | 2026-03-27**

## 【重大新闻】
```

**消息分组**：使用 Markdown 标题层级
```
## 【重大新闻】

**1. 标题**
内容...
来源：来源名

**2. 标题**
内容...
来源：来源名

## 【公司动态】
...
```

### 5. 禁止的做法

❌ **禁止**：
1. 每条新闻单独发送
2. 频繁使用 `---` 分割
3. 附带完整的来源 URL
4. 过度使用表情符号
5. 消息格式不一致

## 📌 示例模板

### 游戏行业日报模板

```
🎮 **游戏行业日报 | YYYY-MM-DD**

## 【重大新闻】

**1. 标题**
新闻内容摘要...
来源：来源名称

**2. 标题**
新闻内容摘要...
来源：来源名称

## 【公司动态】

**3. 标题**
新闻内容摘要...
来源：来源名称

## 【市场与产品】

**4. 标题**
新闻内容摘要...
来源：来源名称
```

### 科技新闻模板

```
💻 **科技速递 | YYYY-MM-DD**

## 【AI动态】

**1. 标题**
内容...
来源：来源名

## 【产品发布】

**2. 标题**
内容...
来源：来源名
```

## ⚠️ 注意事项

1. **统一性**：所有定时任务推送都使用此格式
2. **简洁性**：避免冗余信息
3. **可读性**：优先保证用户阅读体验
4. **来源简洁**：只提供来源名称，不提供 URL
